## P0 Findings

### C-1: MLProjects — Catastrophically Low Test Coverage (0.05 ratio)
- **Repo:** D-sorganization/MLProjects
- **Principle:** PP7
- **Evidence:** ~15K test LOC for 320K source LOC = 0.05 ratio. Target is 0.50. 40 test files exist but are tiny. Many modules have zero tests.
- **Impact:** Any refactor is high-risk. No confidence in releases.
- **Remediation:** 
  1. Add unit tests for all public functions (target: 80% line coverage)
  2. Introduce `hypothesis` property-based tests for math/data modules
  3. Add integration tests for model training pipelines
  4. Block merges below 50% coverage

### C-2: runner-dashboard — Tests Pass Locally but CI Fails
- **Repo:** D-sorganization/runner-dashboard
- **Principle:** PP7
- **Evidence:** 248 tests pass locally, 3 skipped, 1 xfailed. But CI pass rate is 20%. Tests are not the problem — CI environment is.
- **Remediation:** Fix CI environment (Xvfb, timeouts, action versions). See L-1.

### C-3: MEB_Conversion — Only 9 Test Files for 264K LOC
- **Repo:** D-sorganization/MEB_Conversion
- **Principle:** PP7
- **Evidence:** `find tests/ -name "test_*.py" | wc -l` = 9. Core modules (`thermo_database.py`, `gasification_enthalpy.py`, GUI modules) have no tests.
- **Remediation:** Add unit tests for all thermodynamic calculations. Add GUI tests with `pytest-qt` or headless mode.

### C-4: Repository_Management — Low Test Coverage for 580K LOC
- **Repo:** D-sorganization/Repository_Management
- **Principle:** PP7
- **Evidence:** Few test files relative to massive codebase. No coverage reporting.
- **Remediation:** Prioritize tests for modules with highest churn in last 6 months. Add coverage gate to CI.

### C-5: Games — Test Collection Errors (5 errors in 1629 tests)
- **Repo:** D-sorganization/Games
- **Principle:** PP7
- **Evidence:** `pytest --co -q` reports 5 collection errors. Tests exist but cannot be collected.
- **Remediation:** Fix import errors in test files. Add `pytest --co` as a CI gate before running the full suite.
