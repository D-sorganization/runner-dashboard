## P0 Findings

### L-1: runner-dashboard — CI Pass Rate 20% (Catastrophic)
- **Repo:** D-sorganization/runner-dashboard
- **Principle:** PP7
- **Evidence:** Last 20 runs: 16 failures, 3 successes, 1 skipped. Primary failure modes:
  - Xvfb startup errors (`XStartTimeoutError`)
  - Mypy failures
  - Job timeouts (>10 minutes)
  - Jules workflow spam
- **Impact:** No reliable signal on PR quality. Developers ignore CI results.
- **Remediation:**
  1. Pin all GitHub Actions to SHAs or specific versions (no `@main`, no `@v3` floating)
  2. Add Xvfb startup guard: `xvfb-run --auto-servernum` or `pytest-xvfb` plugin
  3. Cap job duration at 10 minutes with `timeout-minutes: 10`
  4. Consolidate Jules workflows into a single `jules-orchestrator.yml`
  5. Add branch protection requiring CI pass before merge

### L-2: Games — Frequent CI Failures with Test Collection Errors
